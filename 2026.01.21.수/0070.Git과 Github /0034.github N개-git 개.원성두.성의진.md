## Git 다중 레포지토리 운용과 Pull의 메커니즘

GitHub에 여러 개의 원격 저장소를 생성하고 PC에도 여러 디렉토리를 만들었을 때, 데이터를 가져오는 방식과 구조에 대해 정리해 드립니다.

---

### 1. 1:1 대응 원칙 (디렉토리와 레포지토리)

Git의 가장 기본적인 원칙은 **하나의 로컬 디렉토리는 하나의 원격 레포지토리와 연결**된다는 것입니다. 따라서 GitHub에 10개의 레포지토리가 있다면, 내 PC에도 이를 담을 10개의 독립된 디렉토리가 필요합니다.

* **개별 관리:** PC의 `Project_A` 폴더는 GitHub의 `Repo_A`와 연결되고, `Project_B` 폴더는 `Repo_B`와 연결됩니다.
* **Pull의 범위:** `Project_A` 폴더 안에서 `git pull`을 실행하면 오직 연결된 `Repo_A`의 내용만 가져옵니다. 한 번의 `pull` 명령으로 GitHub에 있는 모든 레포지토리 내용을 한꺼번에 가져오는 기능은 Git의 기본 설계상 존재하지 않습니다.

---

### 2. 여러 레포지토리를 가져오는 방법 (Clone)

이미 GitHub에 여러 레포지토리를 만들어 둔 상태라면, `pull`이 아니라 **`clone`** 과정을 통해 각각의 디렉토리를 생성해야 합니다.

1. **상위 작업 폴더 생성:** (예: `C:\Work` 또는 `~/Documents/Dev`)
2. **각각 Clone 실행:**
```bash
git clone https://github.com/사용자명/Repo_A.git
git clone https://github.com/사용자명/Repo_B.git
git clone https://github.com/사용자명/Repo_C.git

```


3. **결과:** 상위 폴더 아래에 `Repo_A`, `Repo_B`, `Repo_C`라는 각각의 폴더가 생성되며, 각 폴더는 이미 자신의 `origin` 주소를 알고 있는 상태가 됩니다.

---

### 3. 한 폴더에서 여러 레포지토리를 관리할 수 없나요?

기술적으로 한 폴더에 여러 개의 원격 저장소를 등록하는 것은 가능하지만, 실무에서는 **절대 권장하지 않습니다.**

* **다중 Remote 등록:** 한 폴더에서 `git remote add origin_A [URL_A]`, `git remote add origin_B [URL_B]` 식으로 등록할 수는 있습니다.
* **문제점:** 이럴 경우 소스 코드가 서로 뒤섞이게 되며, 어떤 파일이 어느 레포지토리에 속하는지 구분하기가 매우 힘들어집니다. Git의 이력 관리(Commit History)가 꼬이게 되는 치명적인 원인이 됩니다.

---

### 4. 실무적인 관리 전략

* **독립성 유지:** 프로젝트의 성격이 다르다면 반드시 별도의 폴더로 분리하여 관리하십시오.
* **VS Code 워크스페이스:** 여러 레포지토리를 동시에 보고 싶다면, VS Code의 **"Add Folder to Workspace..."** 기능을 활용하십시오. 물리적으로는 폴더가 분리되어 있어도 에디터 한 화면에서 모든 레포지토리를 제어할 수 있습니다.

---

### [핵심 정리]

1. **Pull**은 현재 내가 위치한 로컬 레포지토리에 연결된 **단 하나의 원격 저장소**에서만 데이터를 가져옵니다.
2. 여러 레포지토리를 가져오려면 각 레포지토리마다 별도의 폴더를 만들고 각각 **`clone`** 해야 합니다.
3. 물리적 폴더는 분리하되, 관리 효율은 VS Code 같은 도구의 기능을 빌려 해결하는 것이 실무 표준입니다.
